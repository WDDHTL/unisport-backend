## ⚠️ 问题与建议

### 🔴 严重问题

#### 1. 敏感信息硬编码
**位置**: `application.properties`
```properties
spring.datasource.password=cjt123456
jwt.secret=UniSportSecretKeyForJWTToken2024SecurityKey
```

**风险**: 数据库密码和 JWT 密钥明文存储在代码库中

**建议**:
```properties
spring.datasource.password=${DB_PASSWORD}
jwt.secret=${JWT_SECRET}
```
使用环境变量或配置中心（如 Spring Cloud Config、Nacos）管理敏感配置。

---

#### 2. CORS 配置过于宽松
**位置**: `WebMvcConfig.java`
```java
registry.addMapping("/**")
    .allowedOriginPatterns("*")  // ⚠️ 允许所有来源
    .allowCredentials(true);
```

**建议**: 生产环境应限制为具体前端域名：
```java
.allowedOriginPatterns("https://unisport.example.com")
```

---

### 🟡 中等问题

#### 3. 命名规范不一致
**位置**: `PostController.java`, `PostServiceImpl.java`
```java
public Result post_Likes(@PathVariable Long id)  // ❌ 下划线命名
public void post_UnLikes(Long id)                // ❌ 不符合驼峰规范
```

**建议**: 统一使用驼峰命名
```java
public Result likePost(@PathVariable Long id)
public void unlikePost(Long id)
```

---

#### 4. N+1 查询问题
**位置**: `PostServiceImpl.java:144-157`
```java
List<PostVO> voList = posts.stream().map(post -> {
    User postUser = userMapper.selectById(post.getUserId());  // ⚠️ 循环内查询
    ...
}).collect(Collectors.toList());
```

**影响**: 若返回100条帖子，将产生100+1次数据库查询

**建议**: 使用批量查询或 JOIN 关联查询
```java
// 方案1: 批量查询用户
Set<Long> userIds = posts.stream().map(Post::getUserId).collect(Collectors.toSet());
Map<Long, User> userMap = userMapper.selectBatchIds(userIds).stream()
    .collect(Collectors.toMap(User::getId, u -> u));

// 方案2: 使用 Mapper XML 编写关联查询
```

---

#### 5. Service 类过大
**位置**: `PostServiceImpl.java` (558行)

**建议**: 拆分为多个专职 Service：
- `PostService` - 帖子 CRUD
- `PostLikeService` - 点赞相关
- `PostNotificationService` - 通知推送

---

#### 6. 缺少分页处理
**位置**: `PostController.java`
```java
@GetMapping
public Result<List<PostVO>> listPosts(@RequestParam Integer categoryId)
```

**建议**: 添加分页参数避免数据量过大
```java
public Result<PageResult<PostVO>> listPosts(
    @RequestParam Integer categoryId,
    @RequestParam(defaultValue = "1") Integer page,
    @RequestParam(defaultValue = "20") Integer size)
```

---

### 🟢 轻微问题

#### 7. 注释编码问题
**位置**: `AuthServiceImpl.java`
```java
// ?????????ID???  // ❌ 乱码注释
```

**原因**: 文件编码不一致（部分 UTF-8，部分 GBK）

**建议**: 统一使用 UTF-8 编码，IDE 配置文件编码为 UTF-8

---

#### 8. 日志使用 System.out
**位置**: `WebSocketServer.java`
```java
System.out.println("客户端 userId=" + uid + " 建立WS连接");  // ❌
```

**建议**: 使用 Slf4j 日志框架
```java
log.info("客户端 userId={} 建立WS连接", uid);
```

---

#### 9. 原始类型 HashMap 使用
**位置**: `PostServiceImpl.java:231`
```java
HashMap map = new HashMap();  // ❌ 原始类型
```

**建议**: 使用泛型
```java
Map<String, Object> map = new HashMap<>();
```

---

#### 10. 魔法值硬编码
**问题**: 状态码、错误消息分散在代码中
```java
throw new BusinessException(40401, "帖子不存在");
throw new BusinessException(40004, "无权限删除");
```

**建议**: 定义错误码枚举类
```java
public enum ErrorCode {
    POST_NOT_FOUND(40401, "帖子不存在"),
    NO_PERMISSION(40004, "无权限操作");
}
```
